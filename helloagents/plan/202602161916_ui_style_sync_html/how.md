# 技术设计：样式同步与 HTML 预览

## 方案概述
- 目标：将 Bitlog 的页面结构套用 `newechoes_git` 的视觉基线，先产出可打开的 HTML 预览文件。
- 输出目录：`docs/ui-style-sync/`。
- 前台范围：`/articles` 列表页、`/articles/:slug` 详情页。
- 后台范围：登录、文章列表、编辑器、设置、账号页。

## 设计要点
1. **视觉基线**  
   以 newechoes 的色板与层级为准：亮色背景、轻投影卡片、蓝色主色、顶部固定导航。
2. **结构对齐**  
   预览 HTML 的结构对齐当前模板/页面结构，避免日后落地偏差。
3. **可复用样式**  
   抽取统一样式到 `docs/ui-style-sync/base.css`，各页面仅保留结构与局部差异。
4. **占位数据**  
   所有数据使用样例内容，避免真实账号/接口/链接。

## 安全与性能
- 安全：不包含任何真实凭证与生产 URL。
- 性能：纯静态 HTML，无运行时依赖。

## 验收方式
- 逐个打开 HTML，检查布局、层级、留白、字体与色彩是否符合 newechoes 风格。
