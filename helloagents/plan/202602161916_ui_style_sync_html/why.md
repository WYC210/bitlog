# 变更提案：同步 UI 样式并输出页面 HTML 预览

## 需求背景
当前 Bitlog 前台与后台的 UI 视觉较旧，整体观感偏“暗黑空洞”，需要以 `newechoes_git` 的视觉风格为基线进行同步，
并先产出每个页面的静态 HTML 预览，便于审阅与确认方向。

## 变更范围
1. 以 `newechoes_git` 作为视觉基线，提炼色板、卡片、导航、按钮等样式规范。
2. 输出每个页面的静态 HTML 预览（前台文章列表/详情；后台登录/列表/编辑/设置/账号）。
3. 预览 HTML 放在 `docs/ui-style-sync/`，仅用于设计审阅，不改动线上模板。

## 影响范围
- 模块：前台 UI / 后台 UI（仅视觉同步与预览）。
- 文件：新增 `docs/ui-style-sync/*.html` 及公共样式文件。
- API/数据：无变更。

## 核心场景
### 需求：样式同步预览
**模块：** 前台 UI / 后台 UI  
在不改动现有代码的前提下，提供可打开的页面 HTML 预览，让 UI 风格先过审。

#### 场景：前台预览
- 期望：文章列表与文章详情的预览页面符合 newechoes 的视觉语言（明亮、卡片化、层级清晰）。

#### 场景：后台预览
- 期望：登录、文章列表、编辑器、设置、账号页的预览页面视觉统一、信息密度合理。

## 风险
- 风险：预览 HTML 与真实组件结构存在偏差。
- 规避：以当前模板/页面结构为基准，仅做视觉样式替换，并在预览中标注占位数据。
