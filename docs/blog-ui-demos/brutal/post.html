用 Cloudflare Worker 部署 Bitlog：从 0 到上线

本文用“静态 demo 内容”模拟真实文章页：标题层级、代码块、引用、列表与链接等。

1. 前置准备

你需要准备 Cloudflare 账户、GitHub 账户，以及一个可用域名。部署完成后建议先访问 workers.dev 域名确认服务可用。

2. 数据库迁移

本地调试时，如果遇到 no such table: settings，通常说明你还没把 migrations 应用到本地 D1。

wrangler d1 migrations apply bitlog --local --cwd apps/api -c wrangler.toml

3. 常见问题

  • 域名不生效：用 nslookup / dig 验证
  • PBKDF2 迭代次数过高会触发 Workers 上限
  • 使用 PASSWORD_PEPPER 增强口令安全性
