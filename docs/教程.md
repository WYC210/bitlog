# 前期准备

你需要准备：

- 一个 Cloudflare 账户
- 一个 Github 账户
- 一个没有被阻断的域名

# 第一步：设置 Github 项目

首先fork[我的仓库](https://github.com/WYC210/bitlog) 到你的账户（要是能顺手点个星星就好啦）进入Action页面，启用Action。
点击 `settings` - `Secrets and variables` - `Actions` 准备添加 `Repository secrets` 接下来我们一共要添加五个secrets。
1.account id
然后进入 Cloudflare 控制台 [https://dash.cloudflare.com/](https://dash.cloudflare.com/)
在浏览器地址栏中复制你的account id ![](https://linux.do/uploads/default/original/4X/a/6/5/a65d50c53bd48ba1bae671f8db1cb262803ad594.png)
回到github页面，点击 `New repository secret` 名称写 `CLOUDFLARE_ACCOUNT_ID` 值就写刚才复制的id。2. CLOUDFLARE_API_TOKEN
接着访问[https://dash.cloudflare.com/profile/api-tokens](https://dash.cloudflare.com/profile/api-tokens) 选择编辑Worker的模板, 然后给它再添加D1的编辑权限,再创建api令牌，回到github页面添加为 `CLOUDFLARE_API_TOKEN` 3. CLOUDFLARE_D1_DATABASE_ID
去Cloudflare控制台的 `存储与数据库` - `D1 sql 数据库` 创建一个名称为bitlog的数据库，并复制它的id。回到github，添加为 `CLOUDFLARE_D1_DATABASE_ID` 4.白嫖免费域名
去[https://domain.stackryze.com/](https://domain.stackryze.com/) 免费白嫖一个域名，有效期是一年，在快过期的时候会发邮箱提醒你，你到时候在续费即可
5.CLOUDFLARE_ROUTE_HOSTNAME 和 CLOUDFLARE_ZONE_NAME
你创建的域名是 `xxx..indevs.in` 就是 `CLOUDFLARE_ROUTE_HOSTNAME` 的值，然后 `CLOUDFLARE_ZONE_NAME` 的值为 `indevs.in`
至此在github上的配置就完毕了，点击上方的`Actions`选项卡，选到Build工作流点运行即可。

# 第二步：建表

去CF的控制台打开刚才创建的数据库，点`Explore Data` 然后在Query窗口中分别粘贴并运行项目中的sql建表语句(定义在 bitlog\apps\api\migrations)，下列我也一一列举出来

```sql
CREATE TABLE IF NOT EXISTS admin_users (
  id TEXT PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  password_hash BLOB NOT NULL,
  password_salt BLOB NOT NULL,
  password_iterations INTEGER NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

```sql
CREATE TABLE IF NOT EXISTS admin_sessions (
  id TEXT PRIMARY KEY,
  admin_user_id TEXT NOT NULL,
  refresh_token_hash BLOB NOT NULL,
  expires_at INTEGER NOT NULL,
  created_at INTEGER NOT NULL,
  last_seen_at INTEGER NOT NULL,
  user_agent TEXT NULL,
  ip TEXT NULL,
  UNIQUE (refresh_token_hash),
  FOREIGN KEY (admin_user_id) REFERENCES admin_users(id) ON DELETE CASCADE
);
CREATE INDEX IF NOT EXISTS admin_sessions_admin ON admin_sessions(admin_user_id, expires_at);
```

```sql
CREATE TABLE IF NOT EXISTS rate_limit_counters (
  key TEXT NOT NULL,
  window_start INTEGER NOT NULL,
  count INTEGER NOT NULL,
  expires_at INTEGER NOT NULL,
  PRIMARY KEY (key, window_start)
);
CREATE INDEX IF NOT EXISTS rate_limit_expires ON rate_limit_counters(expires_at);
```

```sql
CREATE TABLE IF NOT EXISTS assets (
```

```sql
CREATE TABLE IF NOT EXISTS posts (
  id TEXT PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,

  category_id TEXT NULL,

  publish_at INTEGER NULL,
```

第三步 配置域名解析

1. 在CF中选择 domains `Onboard a domain` 输入你之前生成好的域名 `xxx..indevs.in` 默认创建即可
2. 点击创建好的域名 选择 Go to `DNS` to verify your existing records and configure proxy status. 点击DNS跳转
3. 滑倒最底部看到两个NS，这里我们使用NS委托，复制这两个值回到 [https://domain.stackryze.com/](https://domain.stackryze.com/) 然后选择` manage`
4. 在 DNS Configuration 这里把复制过来的两个值填入即可
   接下来等待DNS生效，你可以直接通过你的workers先访问看看是否可行回到你的CF选择workers 点击 `bitlog-web` 这个workers
   直接访问这个地址即可[bitlog-web.yourusername.workers.dev](bitlog-web.yourusername.workers.dev) 通常是这样的
