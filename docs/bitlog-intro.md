# Bitlog：以快捷键为第一特性的博客系统

Bitlog 是一套 **Cloudflare Workers + D1 单库** 的博客系统：`apps/api` 负责数据与能力，`apps/web` 负责前台 SSR + 同域静态 Admin（`/admin/*`）。

下面这份介绍，重点讲“用起来是什么感觉”，而不是从技术细节出发。

---

## 1) 第一特性：快捷键（Web + Admin 全链路）

我把“快捷键”当成 Bitlog 的入口体系：不只是“加几个热键”，而是让**命令面板 / 快捷菜单 / 页面跳转 / 编辑器动作**都可以被快捷键驱动。

### 你会得到什么

- **命令面板**：按一个键，搜索并执行动作（跳转、发布、刷新预览……）
- **快捷菜单（Switch Menu）**：更偏“视觉化”的快速切换入口（支持多种布局）
- **可配置 / 可录制**：支持两种绑定方式
  - 组合键（chord）：如 `mod+shift+p`
  - 序列键（seq）：如 `k k k`（连续按三下 K）
- **输入中也能触发**：编辑器里打字时，像“保存 / 发布 / 刷新预览”这种动作也允许触发（避免“必须先点到空白处才能按快捷键”的割裂感）

### 一些默认手感（你也可以改）

- 前台：`Alt + \`` 呼出快捷菜单；`?` 打开命令面板（移动端也可用）
- 后台编辑器：
  - `mod + s` 保存
  - `mod + shift + p` 发布
  - 也可以把“发布”绑定成 `k k k` 这种序列键

> 快捷键入口不应该是“记不住就算了”，所以命令面板会把可用动作和绑定展示出来，你也可以在后台直接录制。

---

## 2) 导航也是配置：显示 / 隐藏 / 排序 / 自定义

Bitlog 的前台顶部导航不是写死的：

- 支持 **显示/隐藏**（隐藏后该页面会直接返回 404）
- 支持 **调整顺序**
- 支持 **新增/删除自定义项**（可指向站内路径或外链）
- 快捷菜单/命令面板会跟着同步：被隐藏的页面入口与快捷键跳转都会失效

---

## 3) 轻量 SSR + “局部切页”的速度感

前台是服务端渲染（SSR），同时对“首页 / 文章 / 项目 / 工具 / 关于”等核心页面做了 **DOM 级别的局部切换**：

- 点击导航时优先局部更新（不整页闪烁）
- 页面样式与脚本按需加载
- 切页后会自动做必要的 UI 同步（例如导航高亮、搜索框状态等）

---

## 4) 你能开箱即用的页面能力

- **文章**：列表 + 搜索 + 分类/标签过滤 + 文章详情页
- **项目**：GitHub/Gitee 拉取与展示（可配置 Token，服务端保存不暴露给访客）
- **工具中心**：支持“链接型工具”与“页面型工具”（可挂客户端脚本）
- **关于页**：技能 / 足迹 / 经历结构化展示 + 侧边栏模块开关

## 6) 使用建议（最推荐的玩法）

1. 先把前台导航配置成你真正想要的入口（隐藏不需要的页）
2. 再用命令面板把“高频动作”变成肌肉记忆（发布/保存/跳转/新建）
3. 最后用序列键给自己做“私货”：例如把发布绑定成 `k k k`

如果你愿意，我也可以按你现在的工作流（比如“写作/预览/发布”）直接给你配一套默认快捷键方案。
