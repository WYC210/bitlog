文章页不做横向滚动，而是做成一篇有节奏、有沉浸感的阅读界面。

这版 demo 的重点不是花哨，而是把“标题区、正文区、目录轨道、结尾推荐”做成一套更像编辑排版的阅读系统。 about 页负责吸引，文章页负责让人真正读下去。

Article Opening

这类页面本质上是阅读器，不是作品展板。你可以保留你现在已有的 Markdown 渲染能力、目录提取、标签系统和相关文章逻辑， 但视觉表现应该更像一本被认真编辑过的长文，而不是“标题 + 正文 + 侧栏”的默认博客模板。

01

文章头部应该像开篇，而不是一个普通卡片

现在很多博客的标题区都太快进入正文了，读者还没建立预期，页面就已经开始讲内容。更好的做法是先用标题、摘要、分类、时间、 阅读时长这些信息给出一个“阅读入口”，让用户在 3 秒内知道这篇文章在讲什么、值不值得继续读。

如果有封面图，它不应该只是正文里的一张插图，而应该成为标题区的一部分。哪怕没有真实封面，也可以用抽象图形、网格、标语或数据块， 让头图承担情绪和节奏的作用。

Design Note

标题区负责建立期待，正文区负责兑现期待。

02

正文要安静，但不能无聊

正文的核心任务是让眼睛舒服地往下读，所以它不适合横向滚动，也不适合过重的动画。真正有效的是细节层面的排版质量： 宽度、行高、段落间距、标题锚点、代码块、引用块、表格和图片的层次是否清晰。

轻动效应该跟着内容节点，而不是整页晃动

比如当前屏进入视口时，`h2`、引用块、插图说明可以轻微上浮和显现，这种感觉会让页面更有节奏； 但整篇正文如果一直漂、一直推、一直缩放，反而会破坏阅读专注度。

文章页的动画应该像翻书时的空气感，而不是发布会大屏。 
const readingRule = {
  hero: "有辨识度",
  prose: "稳定可读",
  motion: "轻量克制",
  toc: "持续陪读"
};
03

目录不只是目录，它是阅读轨道

右侧目录在你的站里已经是一个很有价值的模块了，但它现在更像“功能列表”，还不够像“阅读轨道”。 如果升级它的视觉和行为，它就可以承担三件事：显示阅读位置、帮助跳转、降低长文焦虑。

目录区应该持续反馈当前阅读进度

目录中的当前章节需要有明显高亮，顶部最好补一个阅读进度数字或者横条，甚至可以显示“还剩 6 分钟”这种信息。 读者在长文里最怕迷路，所以侧栏一定要持续给状态反馈。

04

结尾区域应该让阅读自然延续

很多文章页在正文结束后就突然结束了，或者只放一个“相关文章”。这会让阅读状态戛然而止。更好的做法是把结尾拆成两层： 一层是对当前文章的收束，一层是对下一篇内容的引导。

推荐区做成阅读路线，而不是普通卡片列表

你可以保留现在的相关文章数据逻辑，但视觉上建议更像“继续阅读”而不是“猜你喜欢”。这样能把文章页之间串起来， 让用户产生“下一篇也值得看”的感觉。

05

这套 demo 最终可以怎样迁回你的真实文章页

实现上不需要推翻你现有的文章系统。你的服务端已经会把标题、摘要、标签、目录和相关文章塞进模板里， 所以真实改造时主要是三件事：重做头图区、重做侧栏视觉、补一个轻量滚动脚本。

  • 模板层：提升 `post-header` 的结构密度和层次。
  • 样式层：把正文变成更稳定的阅读版式。
  • 交互层：增加阅读进度、高亮目录、轻量显现。
